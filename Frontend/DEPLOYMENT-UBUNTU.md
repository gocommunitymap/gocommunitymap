# Ubuntu Server Deployment Guide — Next.js SSR (Standalone)

This app uses `output: 'standalone'` in `next.config.js`, which produces a self-contained server bundle in `.next/standalone`.

---

## Requirements

| Tool    | Version                |
| ------- | ---------------------- |
| Ubuntu  | 20.04 / 22.04 LTS      |
| Node.js | 18.x LTS or 20.x LTS   |
| npm     | 9+                     |
| PM2     | latest                 |
| Nginx   | latest (reverse proxy) |

---

## 1. Provision the Server

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git nginx
```

---

## 2. Install Node.js (via NodeSource)

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v   # should print v20.x.x
npm -v
```

---

## 3. Install PM2 (Process Manager)

```bash
sudo npm install -g pm2
pm2 startup   # follow the output instruction to enable PM2 on boot
```

---

## 4. Upload / Clone the Repository

```bash
# Option A — clone from Git
git clone https://github.com/your-org/your-repo.git /var/www/app
cd /var/www/app

# Option B — upload build artefacts directly (see Section 5b)
```

---

## 5. Build the Application

### 5a — Build on the Server

```bash
cd /var/www/app
npm install
npm run build
```

### 5b — Build Locally and Transfer (Recommended for Production)

Run on your local Windows machine:

```powershell
npm run build
```

Then transfer the standalone output to the server:

```bash
# From your local machine (replace USER and SERVER_IP)
scp -r .next/standalone USER@SERVER_IP:/var/www/app/
scp -r .next/static    USER@SERVER_IP:/var/www/app/.next/static
scp -r public          USER@SERVER_IP:/var/www/app/public
```

> **Important:** The standalone bundle does NOT include `public/` or `.next/static/` — you must copy them separately alongside `.next/standalone/`.

---

## 6. Standalone Directory Layout (on server)

After copying, your `/var/www/app` should look like:

```
/var/www/app/
├── server.js              ← Next.js standalone server entry
├── node_modules/          ← minimal bundled dependencies
├── .next/
│   ├── standalone/        ← the main output
│   └── static/            ← must be copied here manually
└── public/                ← must be copied here manually
```

The standalone entrypoint is `.next/standalone/server.js` (generated by Next.js build), **not** the project-root `server.js`.

---

## 7. Environment Variables

Create a `.env.production` (or `.env`) file inside `/var/www/app/.next/standalone/`:

```bash
sudo nano /var/www/app/.next/standalone/.env
```

Example contents:

```env
NODE_ENV=production
PORT=3000
NEXT_PUBLIC_API_URL=https://api.yourdomain.com
# Add all other NEXT_PUBLIC_* and server-side env vars here
```

---

## 8. Start with PM2

```bash
cd /var/www/app/.next/standalone

pm2 start server.js \
  --name "nextjs-app" \
  --env production \
  -- --port 3000

pm2 save
```

Check it is running:

```bash
pm2 status
pm2 logs nextjs-app
```

---

## 9. Configure Nginx as Reverse Proxy

```bash
sudo nano /etc/nginx/sites-available/nextjs-app
```

Paste the following (replace `yourdomain.com`):

```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    # Serve Next.js static assets directly from Nginx (fast)
    location /_next/static/ {
        alias /var/www/app/.next/static/;
        expires 1y;
        access_log off;
        add_header Cache-Control "public, immutable";
    }

    # Serve public folder directly
    location /public/ {
        alias /var/www/app/public/;
        expires 7d;
        access_log off;
    }

    # Proxy all other requests to Next.js standalone server
    location / {
        proxy_pass         http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection 'upgrade';
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_set_header   X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Enable the site:

```bash
sudo ln -s /etc/nginx/sites-available/nextjs-app /etc/nginx/sites-enabled/
sudo nginx -t          # test config
sudo systemctl reload nginx
```

---

## 10. SSL with Let's Encrypt (HTTPS)

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
sudo systemctl reload nginx
```

Certbot auto-renews via systemd timer. Verify:

```bash
sudo certbot renew --dry-run
```

---

## 11. Firewall (UFW)

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
sudo ufw status
```

---

## 12. Common PM2 Commands

| Command                  | Purpose                    |
| ------------------------ | -------------------------- |
| `pm2 status`             | Show all running processes |
| `pm2 logs nextjs-app`    | Tail live logs             |
| `pm2 restart nextjs-app` | Restart after code change  |
| `pm2 reload nextjs-app`  | Zero-downtime reload       |
| `pm2 stop nextjs-app`    | Stop the process           |
| `pm2 delete nextjs-app`  | Remove from PM2            |

---

## 13. Re-deploy (Update)

```bash
# 1. Upload new build artefacts (from local)
scp -r .next/standalone USER@SERVER_IP:/var/www/app/
scp -r .next/static    USER@SERVER_IP:/var/www/app/.next/static
scp -r public          USER@SERVER_IP:/var/www/app/public

# 2. Reload the process (no downtime)
pm2 reload nextjs-app
```

---

## 14. Logs & Monitoring

PM2 writes logs to `~/.pm2/logs/`. To enable log rotation:

```bash
pm2 install pm2-logrotate
pm2 set pm2-logrotate:max_size 50M
pm2 set pm2-logrotate:retain 7
```

View Nginx access/error logs:

```bash
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

## Troubleshooting

| Symptom               | Fix                                                                        |
| --------------------- | -------------------------------------------------------------------------- |
| `502 Bad Gateway`     | Next.js not running — check `pm2 status` and `pm2 logs`                    |
| Static assets 404     | Ensure `public/` and `.next/static/` copied to server alongside standalone |
| Env vars missing      | Add to `.env` inside `.next/standalone/`, then `pm2 reload`                |
| Port 3000 in use      | Change `PORT` in `.env` and update `proxy_pass` in Nginx config            |
| SSL cert not renewing | Run `sudo certbot renew` manually; check `systemctl status certbot.timer`  |
